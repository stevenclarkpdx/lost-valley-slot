Lost Valley Development Workflow
Development Philosophy
Favor improving existing systems over adding new mechanics.
Preserve the expedition/discovery fantasy.
Presentation must never affect game math.
Engine logic remains deterministic and testable.
UI consumes engine results; it does not create them.
Git Workflow
Work directly on main for now.
Before starting work, run git status.
Before completing work:
Run all tests.
Run production build.
Fix any failures before continuing.
Review the git diff before committing.
Commit only project files.
Never commit:
node_modules
dist
generated caches
Push to origin main after a successful commit.
Report:
commit hash
commit message
summary of changes
Implementation Rules
Keep engine and presentation separated.
Prefer extending existing systems over creating parallel ones.
Do not introduce new dependencies unless necessary.
Do not change RTP targets unless explicitly requested.
Ask before introducing new gameplay mechanics.
Version Philosophy

Major milestones should become meaningful commits.

Examples:

v0.4 Game Feel
v0.5 Presentation
v0.6 Fossil Valley Polish

When committing a task:

- Include all intentional source, asset, documentation, configuration, and test changes currently required for the project to run correctly.
- Exclude generated outputs such as dist, node_modules, caches, temporary files, and local environment files.
- Before committing, report any unrelated pre-existing working-tree changes.
- If unrelated changes exist, do not silently include or discard them; ask for confirmation unless they are clearly part of previously completed project work.

Experience Preservation

When tuning RTP:

Never optimize only for mathematical accuracy.

Preserve the emotional purpose of each reward source.

Cluster wins should remain satisfying.

Large natural clusters should never become "fake wins."

Redistribute RTP by adjusting frequency before dramatically reducing memorable payouts.

Protect the emotional identity of:

- large cluster wins (clusters should have satisfying scaling where large clusters are significantly more rewarding than small ones)
- Golden Amber
- Fossil Valley (valleys should be rewarding but not so much that other reward channels suffer or feel insignificant to the player)
- Field Notes milestones

When making tuning recommendations, explain where RTP was taken from and where it was added.
